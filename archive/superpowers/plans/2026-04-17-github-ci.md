# GitHub + CI Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Push the fleet-tracker monorepo to a public GitHub repository and wire up a GitHub Actions CI pipeline with four jobs: check, unit, integration, and e2e.

**Architecture:** Single workflow file (`.github/workflows/ci.yml`) with four jobs. `check`, `unit`, and `integration` run on every push. `e2e` runs only on pushes to `main`, PRs targeting `main`, and manual dispatch — and only after the other three jobs pass. Each job builds `packages/shared` before running its commands because the shared package has no auto-build and its `dist/` is git-ignored.

**Tech Stack:** GitHub Actions, pnpm workspaces, vitest (unit + integration), testcontainers (integration), Playwright (e2e), Docker (docker-compose for e2e PostGIS)

| Fact                      | Value                                                                                                                 |
| ------------------------- | --------------------------------------------------------------------------------------------------------------------- |
| pnpm version              | `10.27.0` (read automatically from `packageManager` in root `package.json` by `pnpm/action-setup@v4`)                 |
| Node version              | 20                                                                                                                    |
| Shared package build      | Must run `pnpm --filter @fleet-tracker/shared build` after install in every job                                       |
| DATABASE_URL for e2e      | `postgres://postgres:password@localhost:5432/fleet_tracker` (matches `docker-compose.yml`)                            |
| API dist entry            | `apps/api/dist/index.js` (`tsconfig.json`: `outDir: "dist"`, `rootDir: "src"`)                                        |

| Playwright report path    | `apps/web/playwright-report/` (Playwright default, no `outputDir` set in `playwright.config.ts`)                      |
| JWT_SECRET min length     | 32 chars (enforced by `config.ts` zod schema)                                                                         |
| test-setup.ts             | Injects all required env vars for unit + integration tests — no GitHub secrets needed for those jobs                  |

## Task 2: Create the CI workflow file

- [ ] **Step 1: Create the `.github/workflows/` directory and `ci.yml`**

Create `.github/workflows/ci.yml` with the following content:

```yaml
name: CI

on:
  push:
  pull_request:
    branches: [main]
  workflow_dispatch:

jobs:
  # ──────────────────────────────────────────────
  # Job 1: Static analysis
  # ──────────────────────────────────────────────
  check:
    name: Check (lint · typecheck · format)
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4
        # reads packageManager field from root package.json — no version pin needed

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build shared package
        run: pnpm --filter @fleet-tracker/shared build

      - name: Format check
        run: pnpm format:check

      - name: Lint
        run: pnpm lint

      - name: Typecheck
        run: pnpm typecheck

  # ──────────────────────────────────────────────
  # Job 2: Unit tests (no external services)
  # ──────────────────────────────────────────────
  unit:
    name: Unit tests
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build shared package
        run: pnpm --filter @fleet-tracker/shared build

      - name: Run unit tests
        run: pnpm test
        # test-setup.ts injects all required env vars — no secrets needed here

  # ──────────────────────────────────────────────
  # Job 3: Integration tests (testcontainers PostGIS)
  # ──────────────────────────────────────────────
  integration:
    name: Integration tests
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build shared package
        run: pnpm --filter @fleet-tracker/shared build

      - name: Run integration tests
        run: pnpm test:integration
        # testcontainers pulls the PostGIS image and manages its own container lifecycle.
        # Docker is pre-installed on ubuntu-latest.
        # test-setup.ts satisfies config.ts validation — no GitHub secrets needed.

  # ──────────────────────────────────────────────
  # Job 4: E2E smoke tests (full stack + Playwright)
  # ──────────────────────────────────────────────
  e2e:
    name: E2E smoke tests
    runs-on: ubuntu-latest
    needs: [check, unit, integration]
    if: |
      github.ref == 'refs/heads/main' ||
      (github.event_name == 'pull_request' && github.event.pull_request.base.ref == 'main') ||
      github.event_name == 'workflow_dispatch'

    env:
      # Consumed by apps/api/src/config.ts (fail-fast on missing/short value)
      JWT_SECRET: ${{ secrets.CI_JWT_SECRET }}
      # Matches docker-compose.yml: user=postgres, password=password, db=fleet_tracker
      DATABASE_URL: postgres://postgres:password@localhost:5432/fleet_tracker
      # NODE_ENV=test mounts the /test/seed-telemetry route (see apps/api/src/app.ts:38)
      NODE_ENV: test
      PORT: 3000
      # Consumed by apps/web/e2e/smoke.spec.ts:4
      API_BASE_URL: http://localhost:3000
      # Consumed by apps/web/playwright.config.ts (baseURL)
      WEB_BASE_URL: http://localhost:5173

    steps:
      - uses: actions/checkout@v4

      - uses: pnpm/action-setup@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build shared package
        run: pnpm --filter @fleet-tracker/shared build

      # ── Start PostGIS ──────────────────────────
      - name: Start PostGIS
        run: docker compose up -d
        # docker-compose.yml has a healthcheck; we wait for it below

      - name: Wait for PostGIS to be ready
        run: |
          until docker compose exec -T db pg_isready -U postgres -d fleet_tracker; do
            echo "Waiting for PostGIS..."
            sleep 2
          done

      - name: Apply database migrations
        run: pnpm --filter api db:migrate

      # ── Build and start API ────────────────────
      - name: Build API
        run: pnpm --filter api build

      - name: Start API server (background)
        run: node apps/api/dist/index.js &
        # API reads JWT_SECRET, DATABASE_URL, NODE_ENV, PORT from the job env block above

      # ── Build and serve web ────────────────────
      - name: Build web
        run: pnpm --filter web build

      - name: Start web preview server (background)
        run: npx vite preview --port 5173 &
        working-directory: apps/web
        # vite preview default port is 4173 — must specify 5173 to match Playwright baseURL

      # ── Healthcheck before Playwright ─────────
      - name: Wait for services
        run: npx wait-on http://localhost:3000/health http://localhost:5173 --timeout 30000

      # ── Playwright ────────────────────────────
      - name: Install Playwright (Chromium only)
        run: npx playwright install --with-deps chromium

      - name: Run E2E smoke tests
        run: pnpm test:e2e

      # ── Artifacts (best-effort) ───────────────
      - name: Upload Playwright report
        uses: actions/upload-artifact@v4
        if: failure()
        with:
          name: playwright-report
          path: apps/web/playwright-report/
          # This artifact may not exist if the failure occurred before Playwright started.
          # upload-artifact@v4 will log a warning but not fail the job in that case.
          retention-days: 7

      # ── Cleanup (always runs) ─────────────────
      - name: Teardown
        if: always()
        run: docker compose down -v
        # -v removes named volumes (pgdata) — safe because CI runners are stateless.
        # Each run starts with a clean slate regardless.
```

## Task 4: Add the GitHub secret

This is a manual step done in the GitHub web UI.

- [ ] **Step 1: Generate a secure value for CI_JWT_SECRET**

```bash
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

Copy the output (64 hex characters — well above the 32-char minimum that `config.ts` enforces).

- [ ] **Step 2: Add the secret to the GitHub repository**

1. Open your repo on GitHub → **Settings** → **Secrets and variables** → **Actions**
2. Click **New repository secret**
3. Name: `CI_JWT_SECRET`
4. Value: paste the hex string from Step 1
5. Click **Add secret**

This secret is only used by the `e2e` job to start the real API server. It has no relation to any production value.

## Task 6: Trigger E2E manually and verify

- [ ] **Step 1: Trigger a manual E2E run via workflow_dispatch**

```bash
gh workflow run ci.yml
```

- [ ] **Step 2: Watch the run**

```bash
gh run watch
```

All four jobs should go green: `check` → `unit` → `integration` → `e2e` (in parallel for the first three, then `e2e` after they pass).

- [ ] **Step 3: If e2e fails, download the Playwright report**

```bash
gh run download --name playwright-report
open playwright-report/index.html
```

Common failure causes and fixes:

- `wait-on` times out → API or web didn't start. Check if `DATABASE_URL` matches docker-compose credentials exactly (`postgres://postgres:password@localhost:5432/fleet_tracker`).
- `seed-telemetry` 404 → `NODE_ENV` is not `test`. Verify the `env:` block in the e2e job sets `NODE_ENV: test`.
- Playwright `baseURL` mismatch → `vite preview` is on wrong port. Confirm `--port 5173` flag is present.
- Artifact not found warning → failure happened before Playwright started. That's fine — the warning is expected per the spec.

## Task 7: Configure branch protection on main

This is a manual step done in the GitHub web UI, done after CI is green.

- [ ] **Step 1: Enable branch protection**

1. GitHub repo → **Settings** → **Branches** → **Add branch ruleset** (or **Add classic branch protection rule**)
2. Branch name pattern: `main`
3. Enable: **Require status checks to pass before merging**
4. Add required checks: `Check (lint · typecheck · format)`, `Unit tests`, `Integration tests`
   - Do not require `E2E smoke tests` — it doesn't run on every push, so requiring it would block non-main-targeted PRs.
5. Enable: **Require branches to be up to date before merging**
6. Save

Expected: pushes directly to `main` will still work (you own the repo), but this signals good practice on a portfolio project.
